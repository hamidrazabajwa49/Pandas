# Pandas Practice & Reference Notebooks

A structured collection of Jupyter notebooks covering the core pandas workflow — from DataFrame creation to advanced cleaning, merging, and reshaping techniques. Built as hands-on practice and as a quick-reference library for common pandas operations.

---

## Repository Structure

```
Pandas
├── data/
│   ├── practicedata.csv      # Full sales dataset (2,338 rows)
│   └── practicedata.xlsx     # Smaller sample of the same schema (15 rows)
├── notebooks/
│   ├── 01_pandas_basics_reference.ipynb
│   ├── 02_dataframe_creation_and_export.ipynb
│   ├── 03_excel_file_exploration.ipynb
│   ├── 04_employee_data_management_and_merging.ipynb
│   ├── 05_data_cleaning_and_aggregation.ipynb
│   └── 06_advanced_pandas_techniques.ipynb
├── outputs/                  # Generated by the notebooks (csv/xlsx/json exports)

└── README.md
```

> **Note:** Notebooks 01, 02, 04, 05, and 06 build their own small toy datasets inline — they run standalone with no external file dependency. Only **03** reads from `data/practicedata.xlsx`.

## Dataset

Both files in `data/` share the same column schema (a retail sales log) but differ in size and serve different purposes:

| File | Rows | Columns | Purpose |
|---|---|---|---|
| `practicedata.csv` | 2,338 | `JE Code, Store, Country, Region, Date, Item, Salesperson, List Price, Actual Price, Discount %` | Full dataset — used for realistic-scale exploration |
| `practicedata.xlsx` | 15 | Same schema | Small sample — used for Excel I/O practice in notebook 03 |

**Data quirks worth knowing (intentional practice material):**
- `Discount %` is stored as a **string** (`"1.25%"`) in the CSV but as a **float** (`0.0125`) in the Excel file — useful for practicing dtype conversion.
- The Excel sample already contains missing values (e.g. `List Price` is blank in some rows) — useful for missing-data handling.
- `Date` is stored in inconsistent formats across rows (`10-06-18`, `4/23/2015`) — useful for date-parsing practice.

## Notebooks Overview

| # | Notebook | Topics Covered |
|---|---|---|
| 01 | **Pandas Basics Reference** | DataFrame/Series creation, inspection (`head`, `info`, `describe`), selection (`loc`/`iloc`), filtering, column modification, sorting, missing values, aggregation, `groupby`, `apply`, merge, concat, indexing, CSV I/O, `value_counts` |
| 02 | **DataFrame Creation & Export** | Building a DataFrame from a dict, descriptive statistics, exporting to Excel/JSON/CSV, reloading to verify round-trip integrity |
| 03 | **Excel File Exploration** | Reading an external `.xlsx` file, shape/columns/size inspection, dtype and null-count overview, summary statistics |
| 04 | **Employee Data Management & Merging** | Column updates (salary increment), missing-value strategies (mean fill vs. linear interpolation), multi-key sorting, grouped aggregation, and all four merge types — **inner, left, right, outer** |
| 05 | **Data Cleaning & Aggregation** | Selection/filtering, sorting, missing-value imputation, derived columns, renaming, `groupby().agg()`, merging in extra data, custom row-wise logic with `apply` |
| 06 | **Advanced Pandas Techniques** | Duplicate detection/removal, value replacement (`replace`), string methods, dtype conversion, multi-condition filtering, `query()`, pivot tables, multi-index DataFrames, exporting cleaned output |

**Suggested reading order:** 01 → 02 → 03 → 04 → 05 → 06 (basics → I/O → real data → relational ops → cleaning pipeline → advanced reshaping).

## Core Concepts Demonstrated

- **Data Inspection** — `head`, `tail`, `shape`, `columns`, `info`, `describe`
- **Selection & Indexing** — label-based (`loc`) vs. position-based (`iloc`), single/multi-column selection, `set_index` / `reset_index`, multi-index DataFrames
- **Filtering** — boolean masks, multi-condition filters (`&`, `|`), the `query()` API
- **Data Cleaning** — null detection (`isnull`), drop (`dropna`) vs. fill (`fillna`) vs. interpolate, duplicate handling, dtype conversion
- **Transformation** — column derivation, renaming, value replacement, string operations, `apply()` with custom/lambda functions
- **Aggregation** — `groupby`, `.agg()` with multiple functions, `value_counts`, `unique`
- **Combining Data** — `merge` (inner/left/right/outer) vs. `concat`, and when each is appropriate
- **Reshaping** — `pivot_table`, multi-index structures
- **I/O** — reading/writing CSV, Excel, and JSON, with round-trip verification

## Setup

**Requirements:** Python 3.9+

```bash
git clone <repo-url>
cd <repo-name>
pip install -r requirements.txt
```

**`requirements.txt`:**
```
pandas
openpyxl
jupyter
```

> `openpyxl` is required for `.xlsx` read/write support in pandas.

## Running the Notebooks

```bash
jupyter notebook notebooks/
```

Run notebooks in order from the repository root so relative paths (`data/...`, `../outputs/...`) resolve correctly. Each notebook is self-contained for its dataset — no notebook depends on another having been run first, except where noted.
